# 电脑文件整理 (File Organizer) 规格说明

## 1. 目标与背景
实现一个智能的、项目优先的电脑文件自动化整理工作流。在不破坏用户现有工作流的前提下，解决桌面或特定下载夹杂乱无章的问题，未来将向全自动化无头定时任务演进。

## 2. 核心原则
- **项目归属优先**：识别文件所属项目是最优先的整理逻辑，格式与时间兜底。
- **人工复核至上**：在全自动模式成熟前，AI 必须生成规划并获取用户的显式确认后才能执行文件移动。
- **保护现有结构**：已经是“项目”的文件夹，只深入整理其内部结构，不改变项目文件夹本身的层级和位置。

## 3. 工作流程设计

### 阶段 1：扫描与智能规划
1. **探索上下文**：Agent 读取目标目录（用户动态指定），提取第一层文件和目录。已存在的目录默认视为“已知项目”（Active Projects）。
2. **AI 推断分类**：
   - 若文件名具有明显项目聚集特征或匹配现有项目，则归入该项目。
   - 若发现新的文件聚集特征，拟定新项目名称（作为建议）。
   - 无法判定项目归属的零散文件，标记为“混合兜底分类”。
3. **输出复核计划**：在聊天中向用户展示《分类执行计划》。**必须停顿，等待用户确认**。

### 阶段 2：执行与收尾
1. **获取授权**：用户回复“确认”或给出微调意见。
2. **脚本执行**：调用 `organizer.py` 脚本进行物理移动。
3. **安全网逻辑（脚本内置）**：
   - 执行“混合兜底分类”时，按扩展名归类（Images, Documents, Apps 等）。
   - 若目标分类文件夹内部文件数 > 50，则自动按文件的最后修改时间创建子文件夹（如 `2026-04`）。
4. **最终汇报**：总结处理的文件数及最终目录结构。

## 4. 文件与组件架构
- `SKILL.md`: 存放给 Agent 识别的工作流和复核红线。
- `scripts/organizer.py`: 负责执行底层文件移动和安全网数量判断的轻量级 Python 脚本。

## 5. 局限与未来演进
目前需要人工复核。待规则稳定且准确率达到要求后，可剥离人工复核步骤，由 cron 工具挂载为后台静默定期的整理任务。
