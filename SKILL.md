# File Organizer (电脑文件整理)

本技能用于帮助用户智能化地整理凌乱的文件目录。核心思想是**“项目优先，兜底归类”**。

## ⚠️ 绝对规则 (HARD-GATE)

**绝不允许在未经用户同意的情况下直接移动文件！**
必须严格按照“扫描 -> 规划 -> 请求确认 -> 执行”的顺序操作。

## 步骤 1：扫描目标目录
当用户要求整理某个目录（例如 `~/Downloads`）时：
1. 首先使用 `terminal` 列出该目录下的所有项 (`ls -1p`)。
2. 以 `/` 结尾的是目录，将其视为**已有的项目/分类**。
3. 没有 `/` 的是散文件，这些是我们本次需要整理的对象。

## 步骤 2：智能规划 (AI 大脑)
对于发现的散文件，在脑海中（或在暂存区）生成一份《分类执行计划》。判断逻辑：
1. **项目优先**：如果文件名带有某个特定客户名、活动名，或者明确从属于第一步扫描出的现有项目目录，把它划分为该项目。如果有明显的新的文件聚集特征（如很多装修相关的单据），拟定一个新的项目名称。
2. **混合兜底**：如果文件看起来就是毫无关联的单体文件（如孤立的一张网页截图、一个下载的软件安装包），把它归类为“兜底”(Fallback)。

## 步骤 3：申请复核 (请求确认)
在聊天窗口中，以清晰的格式向用户展示你的《分类执行计划》。
格式示例：
```text
【项目归类】
- 归入 "2026-报销项目": receipt_1.pdf, invoice.jpg
- 归入 "装修资料" (建议新建): 户型图.png, 报价单.xlsx

【格式兜底归类】
- 图片: screenshot.png
- 压缩包: node_modules.zip
```
**停下！** 询问用户：“老板，这个分类计划没问题吗？回复确认我才执行。”

## 步骤 4：生成 JSON 并执行脚本
用户确认后：
1. 在临时目录（如 `/tmp/organize_plan.json`）生成 JSON 格式的执行计划。JSON 格式必须严格如下：
```json
{
  "project_moves": [
    {"file": "receipt_1.pdf", "project": "2026-报销项目"},
    {"file": "户型图.png", "project": "装修资料"}
  ],
  "fallback_moves": [
    "screenshot.png",
    "node_modules.zip"
  ]
}
```
2. 调用底层脚本执行：
```bash
python3 /Users/robinlu/Self-established_skill/file-organizer/scripts/organizer.py <目标目录> /tmp/organize_plan.json
```
3. 脚本执行完毕后，将脚本的输出结果反馈给用户。

> **底层脚本机制科普（Agent了解即可）：**
> 脚本 `organizer.py` 会将 `fallback_moves` 里的文件按照固定字典移动到 Images/Documents/Apps/Media/Archives 下。如果目标文件夹内的文件数量超过 50 个，脚本会自动读取该文件的修改时间，创建一个诸如 `Images/2026-04/` 的月份子文件夹再放进去。
