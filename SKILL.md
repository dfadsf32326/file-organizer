# 电脑文件智能整理 (File Organizer)

## 概述
实现一个智能的、项目优先的电脑文件整理工作流。在不破坏用户现有工作流的前提下，解决桌面或特定文件夹杂乱无章的问题，支持“AI大脑规划”与“底层脚本执行”的混合分离模式。

## 核心原则

1. **项目归属优先**：识别文件所属项目是最优先的整理逻辑，优先于文件格式或创建时间。
2. **人工复核至上（防手残机制）**：AI 必须先输出清晰的规划清单，在得到用户明确回复“确认”之前，绝对不能执行任何移动文件的操作。
3. **保护现有结构**：已经是明确项目或分类的子文件夹，只深入整理其内部结构，不改变文件夹本身的层级。
4. **格式与时间兜底**：对于非项目文件，按扩展名归类。若目标文件夹（如 Images）内部文件数 > 50，自动按文件的最后修改时间创建子文件夹（如 `2026-04`）。

## 工作流程

### 阶段 1：扫描与智能规划（大脑）
1. **获取目标路径**：确认用户要整理的目标目录（支持自然语言或明确路径）。
2. **扫描上下文**：使用 terminal 或相关工具列出第一层文件和目录。现存的目录可作为判断活跃项目的依据。
3. **推断项目归属**：
   - 提取文件名聚类特征。
   - 对无明显特征的文件使用 `read_file` 抽查内容判定。
   - 形成“属于已知项目”、“拟定新项目”或“混合兜底（无项目）”三种分类。
4. **提交复核计划**：**在聊天中清晰列出《分类执行计划》。必须停顿并明确等待用户指令。**

### 阶段 2：执行与收尾（肌肉）
1. **获取授权**：等待用户确认。
2. **调用脚本**：执行底层的 Python 脚本 `scripts/organizer.py`。
3. **输出汇报**：向用户汇报移动成功的文件数和最终的目录树状态。

## 常见错误（反模式）

| 错误 | 后果 | 应对方案 |
|------|------|----------|
| 未经复核直接移动文件 | 打乱用户现有工作流，重要文件丢失或错位 | 必须有明确的 `wait for user confirmation` 步骤 |
| 仅按格式(如.jpg)分类 | 破坏了“项目”的聚合性，属于同一项目的设计图和文稿被拆散 | 执行第一步的 AI 智能项目推断，找出聚类特征 |
| 文件夹内堆积几千个文件 | 电脑卡顿，查找困难 | 脚本执行兜底分类时，加入超 50 个文件自动按月划分子文件夹的逻辑 |

## 未来演进
当该技能的 AI 分类准确率和稳定性达到预期后，可剥离“人工复核”阶段，利用 Cron 工具实现完全无头（Headless）的后台定期自动整理。
